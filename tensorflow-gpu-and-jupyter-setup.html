AWS Instance Setup and TensorFlow with GPU Support



These are my notes on setting up an AWS instance with GPU, installing TensorFlow with GPU support, and Jupyter as a service on the AWS instance.



Set up AWS Instance


We need to find a way to share the instance image on AWS, or create a docker instance from the AWS instance that people can use. For now, these are the steps we took to set up an AWS instance with a GPU so that we can run TensorFlow on the GPU.


Here are the steps, from clicking on the, “launch instance,” button in the AWS EC2 console.


These steps are a sketchy outline of how to set up the AWS instance. If the reader is unfamiliar with setting up AWS instances, please do a Google search on how to set up AWS instances for more details. Each step is something we clicked on in each screen of the launch instance flow in the AWS console.


  • Select ubuntu instance

  • Select g2.2xlarge instance

  • No change on configure instance details

  • Change root partition to 64 GB root storage

  • Create Name tag / use any name


The security group and key pair will differ for the readers’ AWS accounts. These are what we used:


  • Use sg-fddb5995 quick-start-1 security group

  • Use aws-tensorflow key pair


After setting up the AWS instance, one must take the prerequisite step of installing Nvidia’s CUDA toolkit, and the cudNN libraries. Tensorflow has a dependence on this software.


When taking these steps, we advise the reader to browse to https://www.tensorflow.org/install/install_linux . This document has more details on how to do the install, as well as other installation paths that might be more desirable to take.


Hint: to see the version of Ubuntu that your AWS instance is running, use the command: cat /proc/version


Hint: to double check that your AWS instance has a Nvidia graphics card use the command: lspci | grep -i nvidia


Install kernel headers with the command:

sudo apt-get install linux-headers-$(uname -r)


Note: for us, the kernel headers were already installed.


Choose installation method: open a browser to: https://developer.nvidia.com/cuda-downloads .




One must click on several buttons on the downloads page to get a screen like the above. We clicked:

  • linux operating system

  • x86_64 architecture

  • Ubuntu distribution

  • Version 16.04, deb (network) install type


We download the .deb by clicking button on web page, and upload it to the AWS instance we created above.


Next SSH to the AWS instance, and execute commands similar to the

use base installer instructions from nVidia web page:


  • sudo dpkg -i cuda-repo-ubuntu1604_8.0.61-1_amd64.deb

  • sudo apt-get update

  • sudo apt-get install cuda


Note: the Debian package that the reader would install in the steps above may differ. The difference we would owe to a package for a different Ubuntu version than the above.


Reboot the AWS instance.


If the reader is following the installation documentation on the Nvidia web site, the reader may skip down to the post installation actions

We then took the step: add lines to ~/.bashrc:

# add cuda toolkit to path


[[ ":$PATH:" != *":/usr/local/cuda-8.0/bin:"* ]] && PATH="/usr/local/cuda-8.0/bin:${PATH}"


Note the line of bash scripting language above will add the CUDA toolkit executables directory to the PATH environment variable if that variable does not already contain the directory path.


We found it was not necessary to do commands regarding LD_LIBRARY_PATH when using this installation path in the post installation steps in the Nvidia documentation.


Install cuDNN library

Open a browser to https://developer.nvidia.com/cudnn.


Click download. Click log in.



Click Download cuDNN v6.0 (April 27, 2017), for CUDA 8.0.


Click cuDNN v6.0 Runtime Library for Ubuntu16.04 (Deb)


Download libcudnn6_6.0.21-1+cuda8.0_amd64.deb


Upload the .deb file to the AWS instance:


SSH to the AWS instance, and install the Debian package:


sudo dpkg -i libcudnn6_6.0.21-1+cuda8.0_amd64.deb


Install the NVIDIA CUDA Profile Tools Interface


At this point, we return to the TensorFlow documentation at https://www.tensorflow.org/install/install_linux


Execute the command: sudo apt-get install libcupti-dev


We are finished with TensorFlow setup prerequisites.


Install TensorFlow in a virtual environment:

Open a browser to: https://www.tensorflow.org/install/install_linux#InstallingVirtualenv


We use python 2.7 commands:

sudo apt-get install python-pip python-dev python-virtualenv # for Python 2.7


Ccreate virtual env directory


mkdir tensorflow-pip-install


Create virtual environment:


virtualenv --system-site-packages tensorflow-pip-install # for Python 2.7


Aactivate the virtual environment:


source ~/tensorflow-pip-install/bin/activate


Ensure we have installed the latest pip


easy_install -U pip


Install tensorflow with gpu support:


pip install --upgrade tensorflow-gpu # for Python 2.7 and GPU


Note: the commands to do this are located in a script at: https://raw.githubusercontent.com/jhancock1975/tensorflow-chatbot-tutorial/master/src/virtual-env-setup.sh


Install Jupyter


Open a browser to http://www.albertauyeung.com/post/setup-jupyter-nginx-supervisor/ . This document has details on how to do this setup. Note: the reader should do these commands in an SSH session to the AWS instance.


Install Jupyter in virtual environment


pip install Jupyter


Validate install


jupyter notebook

Note: should see terminal based browser with jupyter home page


Create Jupyter configuration


In a Python virtual environment execute the command:


jupyter notebook –generate-config this will create a Jupyter configuration in the reader’s home directory, under the .jupyter subdirectory. The file name is: upyter_notebook_config.py.


The reader can copy and paste the the lines:


c.NotebookApp.base_url = '/notebook'

c.NotebookApp.password = 'sha1:2828a54816ae:14581c8b67e5dedc9b407f83d360300ae3155dda'

c.NotebookApp.port = 8888

c.NotebookApp.port_retries = 50


Into that file, or use the one we have in version control at:


https://github.com/jhancock1975/tensorflow-chatbot-tutorial/blob/master/files/jupyter_notebook_config.py


Install Jupyter as a Service


Create supervisor configuration file for Jupyter service:


Use file at https://github.com/jhancock1975/tensorflow-chatbot-tutorial/blob/master/files/jupyter.conf


Upload this file to the AWS instance, then login to AWS instance, copy config file, create directory for logging etc., reload and update supervisor


Run the following commands:


mkdir /home/ubuntu/supervisor-jupyter

touch /home/ubuntu/supervisor-jupyter/jupyter-service.log


sudo cp jupyter.conf /etc/supervisor/conf.d/


sudo supervisorctl reread

sudo supervisorctl update


Jupyter should be available at something like:

http://ec2-54-158-66-128.compute-1.amazonaws.com/notebook
